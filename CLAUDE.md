## 2. Project Vision

A **dead-simple, fully open-source** homelab deployment tool that beats SimpleHomelab/Deployrr
on ease-of-use while staying 100% free (no paid tiers, no PHP, no Cloudflare dependency).

**Core principles:**
- Single `curl | sudo bash` install — zero manual steps
- Pure Bash TUI + beautiful Flask WebUI — no PHP, no Node, no NPM
- 100+ apps supported across 15+ categories
- Everything self-contained in the Docker image
- MIT licensed, fork-friendly, every line documented

**We are NOT:**
- Copying Deployrr's PHP code or business model
- Requiring Traefik (optional, not mandatory)
- Building a paid tier
- Adding Proxmox management (pure Docker homelab focus)

## 4. Architecture Decisions (FINAL — Do Not Change Without Updating This File)

### 4A. App Catalog Format
**Decision: Hybrid `catalog.json` + `catalog.sh`**

```
apps/
├── catalog.json     ← parsed by app.py for WebUI
└── catalog.sh       ← sourced by deployrr.sh for TUI (auto-generated from catalog.json)
```

**Rationale:**
- `catalog.yaml` rejected because it requires `yq` runtime dep on host
- `catalog.json` is parsed natively by Python (zero deps)
- `catalog.sh` is generated by a script — adding an app = edit catalog.json only
- No YAML parser needed in Bash

### 4B. Authentication
**Decision: Single bearer token + setup guide**

- First run: `app.py` generates a random 32-char token, prints to container logs
- User sets `DEPLOYRR_TOKEN` env var to override with custom token
- WebUI login page: enter token → stored in `localStorage`
- Every API call sends `Authorization: Bearer <token>` header
- Setup guide (in README + CLAUDE.md) covers: changing token, putting it behind Nginx/HTTPS

**Auth is OPTIONAL for local LAN use.** If `DEPLOYRR_TOKEN` env var is not set and no token has been generated, auth is bypassed (dev mode). This is documented clearly.

### 4C. Real-Time Updates
**Decision: SSE (Server-Sent Events) for dashboard, WebSocket for log streaming**

- Replace all `setInterval` polling with SSE stream at `/api/stream`
- SSE pushes overview data every 2 seconds
- WebSocket (`flask-sock`) used only for live log tailing
- Rationale: SSE works through proxies, zero client library needed, simpler than full WebSocket

### 4D. State Persistence
**Decision: SQLite at `/data/deployrr.db` inside container**

Tables:
- `settings` — key/value (config_dir, media_dir, tz, puid, pgid, token_hash)
- `deploy_history` — timestamp, apps deployed, status, compose_snapshot
- `alerts` — threshold config per metric

Volume mount: `-v /opt/deployrr/data:/data`

### 4E. Compose Generation
**Decision: Snippet injection (not full-file rewrite)**

Each app has a compose snippet in `catalog.json`. The deploy function:
1. Reads current `docker-compose.yml`
2. Injects new service snippet
3. Writes back
4. Runs `docker compose up -d <new_service>`

This enables Stack Manager (add/remove without touching existing services).

### 4F. No External Build System
**Decision: Vanilla JS + CSS in single `app.py` HTML string**

- No NPM, no Webpack, no React build step
- Fonts from Google Fonts CDN (already in place)
- All JS is vanilla ES2020 (modules syntax where needed)
- Rationale: The whole point is zero-dep simplicity

## 8. Auth Setup Guide

### Default Behavior
When the WebUI container starts for the first time with no `DEPLOYRR_TOKEN` set:
1. A random 32-character token is generated
2. It is printed to the container logs:
   ```
   docker logs deployrr_webui | grep "DEPLOYRR TOKEN"
   ```
3. The WebUI shows a login screen requiring this token
4. The token is stored (hashed) in SQLite

### Setting a Custom Token
```bash
# Option 1: Environment variable (recommended)
docker run -e DEPLOYRR_TOKEN="your-secret-token" ...

# Option 2: In docker-compose.yml
environment:
  - DEPLOYRR_TOKEN=your-secret-token

# Option 3: After install, edit /opt/deployrr/data/.env
echo "DEPLOYRR_TOKEN=your-secret-token" >> /opt/deployrr/data/.env
```

### Disabling Auth (LAN-only, trusted network)
```bash
docker run -e DEPLOYRR_NO_AUTH=true ...
```
**Warning:** Only do this if the port is not exposed to the internet.

### Putting Deployrr Behind HTTPS (Production)
See `docs/auth-setup.md` for full Nginx and Caddy examples.

**Quick Nginx snippet:**
```nginx
server {
    listen 443 ssl;
    server_name deployrr.yourdomain.com;
    ssl_certificate /etc/ssl/certs/your.crt;
    ssl_certificate_key /etc/ssl/private/your.key;

    location / {
        proxy_pass http://localhost:9999;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
        # Required for SSE:
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 3600;
    }
}
```

**Important for SSE:** `proxy_buffering off` is mandatory, otherwise live dashboard updates won't stream.

## 12. Conventions & Standards

### Bash (deployrr.sh)
- All functions prefixed by context: `d_` (dialog), `log_` (logging), `add_service_` (compose writer)
- Every function has a one-line comment above it
- `set -uo pipefail` at top — strict mode always on
- Error handling: every docker command followed by exit code check + `log_err`
- No hardcoded paths — always use `${CONFIG_DIR}`, `${MEDIA_DIR}`, `${SCRIPT_DIR}`

### Python (app.py)
- All API routes prefixed `/api/`
- All helper functions prefixed `_` (private)
- Error responses always return `{"error": "message"}` with appropriate HTTP status
- Every route has a one-line docstring
- SQLite access via context manager (`with sqlite3.connect(DB_PATH) as conn:`)
- HTML is in a single `HTML = r"""..."""` string at the bottom of the file

### JSON (catalog.json)
- All IDs lowercase, underscores for spaces (`home_assistant` not `homeAssistant`)
- All images use specific tags where known, `:latest` only when no stable tag exists
- `full_stack: true` means this app is included in the "Full Stack" preset
- Every app must have a `description` (one sentence, max 100 chars)
- Ports in `"HOST:CONTAINER"` format, always strings

### Git
- Commits: `Stage N: brief description` for staged work
- Tags: `vMAJOR.MINOR.PATCH` (semver)
- Branch: `main` only (no dev branch needed at personal scale)
- Never commit: `*.db`, `*.log`, `.env`, `data/`
